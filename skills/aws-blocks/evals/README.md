# AWS Blocks Skill — Behavior Evals

This directory holds the **behavior evals** for the `aws-blocks` skill, plus the fixtures and
documentation needed to run them in an A/B benchmark (with skill vs. without skill).

## Purpose

The A/B benchmark answers one question: **does having the skill actually help?**

It runs each eval prompt against a disposable sandbox, twice — once where the agent has access to
`skills/aws-blocks/` (with_skill) and once where it has no skill context (without_skill) — then
compares pass rates, latency, and token usage across multiple runs.

## Directory layout

```
evals/
  evals.json              — eval definitions: prompts, fixtures, expectations
  fixtures/
    blank-ifc/index.ts    — minimal IFC with no data Blocks (eval 0, 1)
    todos-seeded/index.ts — IFC with a todos DistributedTable already defined (eval 2)
  README.md               — this file
  README.ja.md            — Japanese translation
```

Benchmark output (generated by skill-creator tools, not committed):

```
<workspace>/iteration-N/
  eval-0/
    with_skill/run-1/{transcript.md, outputs/, grading.json, timing.json}
    without_skill/run-1/{...}
  eval-1/ ...
  eval-2/ ...
  benchmark.json
  benchmark.md
```

## Fixtures

Each eval specifies a `fixture` field in `evals.json`. Before each run the executor copies `my-app`
to a temp sandbox and overwrites `aws-blocks/index.ts` with the fixture file. This ensures a
reproducible starting state regardless of how the live project evolves.

| Fixture | IFC state | Used by |
|---|---|---|
| `blank-ifc` | Scope + AuthBasic + empty ApiNamespace; no data Blocks | eval 0, 1 |
| `todos-seeded` | Full todos DistributedTable already present | eval 2 |

## Expectations (grading criteria)

Each eval has an `expectations` array — a list of plain-language assertions the grader checks
against the executor's transcript and outputs. The grader returns `grading.json` with a pass/fail
per assertion and an overall score.

See `evals.json` for the current assertions for each eval.

## Running the benchmark

Use the `/run-evals` command, which automates the full sandbox → execute → grade → aggregate flow.
See `.claude/commands/run-evals.md` for the step-by-step runbook.

The benchmark tooling lives in the skill-creator plugin:

```
~/.claude/plugins/marketplaces/anthropic-agent-skills/skills/skill-creator/
```

Quick-start (smoke test — eval 2 only, 1 run per config):

```sh
# from the repo root
claude /run-evals --eval 2 --runs 1
```

Full benchmark (3 evals × 2 configs × 3 runs):

```sh
claude /run-evals --runs 3
```
